# Правила работы с репозиторием GitHub

Добро пожаловать в проект! Пожалуйста, соблюдайте следующие правила при работе с кодом.

## 1. Клонирование репозитория
Для начала работы клонируйте репозиторий на свой компьютер:
```bash
git clone https://github.com/Lynx80/dz.git
cd dz
```

## 2. Работа в ветках (Ветвление)
Не делайте изменения напрямую в ветку `main`. Для любой новой задачи или фикса бага создавайте новую ветку:
```bash
git checkout -b feature/имя-вашей-задачи
```
*(Например: `git checkout -b feature/add-new-button`)*

## 3. Сохранение изменений (Коммиты)
Сохраняйте свои изменения логичными порциями и пишите понятные сообщения к коммитам:
```bash
git add .
git commit -m "Добавлена новая кнопка в меню"
```

## 4. Отправка изменений (Push)
После того как вы сделали коммит, отправьте вашу ветку на сервер GitHub:
```bash
git push origin feature/имя-вашей-задачи
```

## 5. Создание Pull Request (PR)
1. Зайдите в репозиторий на GitHub.
2. Нажмите кнопку **Compare & pull request**.
3. Опишите, что было сделано в вашем PR.
4. Дождитесь проверки (ревью) от владельца репозитория или других участников.
5. Только после одобрения код будет слит (Merge) в главную ветку `main`.

## 6. Синхронизация данных
Если кто-то другой внес изменения в `main`, обязательно скачайте их перед тем, как создавать новые ветки:
```bash
git checkout main
git pull origin main
```
